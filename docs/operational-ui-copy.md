# Regla de contenido para pantallas operativas

Las pantallas operativas deben mostrar únicamente la información y las acciones necesarias para completar la tarea.

## Reglas

- Usar etiquetas breves y datos concretos.
- Priorizar estado, cantidad, unidad, origen, destino y acción.
- No incluir explicaciones técnicas, razonamientos del sistema ni texto conversacional.
- No mostrar nombres internos de tablas, políticas, conversiones, snapshots o implementaciones.
- Reservar explicaciones de configuración para pantallas administrativas.
- Mantener advertencias únicamente cuando indiquen un problema o una acción requerida.
- Evitar mensajes de éxito que expliquen lo que el sistema hará cuando el dato ya sea evidente.

## Ejemplo

Correcto:

- Solicitud: 3 Paquete x6
- Equivalencia: 18 unidades

Incorrecto:

- Modo: Paquete x6
- Se solicitará por cantidad real; no requiere presentación fija.
